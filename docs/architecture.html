Automation Workflows
5 core automated workflows triggered by chatbot commands or direct API calls.
Workflow 01
📄 Document Processing
Extracts structured data from invoices, contracts, and reports using OCR + AI.
1
User uploads PDF via dashboard or email
2
FastAPI saves file, dispatches Celery task
3
pdfplumber extracts text; OCR for scanned PDFs
4
AI detects category (invoice / contract / form)
5
LLM extracts structured JSON fields
6
Data stored in PostgreSQL extracted_data table
7
Dashboard updates; chatbot confirms completion
Workflow 02
🛡️ Security Scan
Full website vulnerability scan: headers, SSL, ports, tech detection, vuln checks.
1
User provides URL via chat or API
2
Task queued in Redis security queue
3
Check 6 security headers (HSTS, CSP, X-Frame…)
4
Analyze SSL certificate validity and config
5
Scan 14 common ports for exposure
6
Detect CMS, server, and frameworks
7
Calculate security score 0–100 with grade A–F
8
AI generates plain-language executive summary
Workflow 03
📋 Report Generation
Professional security audit and document summary reports in PDF format.
1
User requests report from dashboard or chat
2
Worker fetches scan or document data from DB
3
AI writes executive summary in plain language
4
ReportLab builds structured PDF with findings
5
PDF saved to file storage, path stored in DB
6
Download link sent to user dashboard
Workflow 04
📧 Email Automation
Monitors inbox, auto-detects invoices, extracts data, and builds accounting reports.
1
Celery Beat polls IMAP inbox every 5 minutes
2
Detect unread emails with attachments
3
Identify invoice/receipt attachments by name + type
4
Download and save attachment to storage
5
Dispatch to document_worker for AI extraction
6
Mark email as read; store record in DB
Workflow 05
🤖 Chatbot Routing
The chatbot acts as the central interface — detecting intent and triggering the right workflow automatically.
1
User types message in chat
2
Intent classifier detects: scan / document / report / dashboard
3
Extract entities (URLs, file references, IDs)
4
Dispatch to correct Celery task queue
5
Return confirmation with task ID to user
6
Fallback to direct LLM answer for general questions

Deployment Guide
From local dev to production in 5 steps.
📦 Step 1 — Clone & Configure
# Clone and set up environment
git clone <your-repo>
cd cyberplatform
cp .env.example .env

# Fill in .env:
GROQ_API_KEY=gsk_...
DB_PASSWORD=yourpassword
SECRET_KEY=random-256bit-key
🐳 Step 2 — Build & Start
# Build all containers
docker compose build

# Start full platform
docker compose up -d

# Check all running
docker compose ps
🗄️ Step 3 — Database
# Schema auto-applied on first start
# Or manually:
docker exec -it cyberplatform_db psql \
  -U cpuser -d cyberplatform \
  -f /docker-entrypoint-initdb.d/01_schema.sql
✅ Step 4 — Verify
# API health check
curl http://localhost:8000/api/health

# Access points:
API:      http://localhost:8000/api/docs
Dashboard:http://localhost:3000
Flower:   http://localhost:5555
